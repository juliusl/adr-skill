## Stage 1: Add Auto-Commit Support to SKILL.md

### Task 1.1: Add auto-commit prompt to Step 5    [small]

**Description:** Extend Step 5 (Participation Check) in
`src/skills/implement-adr/SKILL.md` to include a second prompt after the
participation mode selection. The prompt asks whether to enable auto-commit
on task completion, with "No" as the default. Also add logic to check for an
existing auto-commit preference from `.meta/` (loaded in Step 0) and apply it
silently if found.

**Implementation Notes:**

Insert after the existing Step 5 sub-step 4 ("Offer to persist"), before the
"Participation Mode Behaviors" heading:

```markdown
5. **Auto-commit preference:** After determining participation mode, check
   whether to enable automatic git commits at task boundaries.
   - **If a `.meta/` policy exists** for auto-commit, apply it silently:
     > Auto-commit on task completion: **enabled** (from .meta policy).
   - **If no preference exists**, prompt:
     > Would you like to create a git commit each time a task completes?
     > 1. **Yes** — Commit after each task's acceptance criteria are all satisfied
     > 2. **No** (default) — I'll manage commits myself
   - **Offer to persist** — same fallback chain as participation mode (if
     `.meta/` exists, offer to save; otherwise session context only).
   - Meta-ADR title pattern: "Auto-commit on task completion: [enabled/disabled]"
```

**Test & Acceptance Criteria:**
- [x] Step 5 in SKILL.md includes a sub-step for auto-commit preference
- [x] The prompt text matches ADR-0010 Decision §4
- [x] Silent application from `.meta/` policy is documented
- [x] Meta-ADR title pattern is specified

**Dependencies:** None

**ADR Reference:** ADR-0010, Decision §4

### Task 1.2: Add auto-commit behavior and guard rails section    [medium]

**Description:** Add a new section to `src/skills/implement-adr/SKILL.md`
documenting the auto-commit behavior: what happens when all checkboxes for a
task are marked, the commit message format, guard rails for unrelated changes,
merge conflicts, pre-commit hook failures, and the Autonomous mode fallback.

**Implementation Notes:**

Add a new subsection `#### Auto-Commit on Task Completion` after the
"Weighted Mode — Per-Task Evaluation" subsection. Include:

1. **When it triggers:** After all `- [ ]` checkboxes in a task's Test &
   Acceptance Criteria are marked `- [x]`.

2. **Commit steps:**
   - Stage the plan file
   - Stage implementation files (tracked by what the agent touched)
   - Create commit with conventional message format
   - Do not push

3. **Commit message format:**
   ```
   implement(<adr-range>): complete task <N.M> — <task title>

   Plan: <plan-file-path>
   Task: <N.M> <task title> [<cost>]
   ADR: <adr-reference>
   ```

4. **Guard rails:**
   - Unrelated changes: warn and ask (Autonomous fallback: commit only
     task-related files, log warning)
   - Merge conflicts / dirty state: pause and ask
   - Pre-commit hook failures: always pause regardless of mode
   - Git state warning for users with custom staging workflows

5. **Interaction table** with all four participation modes

**Test & Acceptance Criteria:**
- [x] SKILL.md contains an "Auto-Commit on Task Completion" subsection
- [x] Commit message format from ADR-0010 Decision §3 is documented
- [x] All four guard rails from ADR-0010 Decision §3 are present
- [x] Autonomous mode fallback for unrelated changes is specified
- [x] Pre-commit hook failure behavior (always pause) is specified
- [x] Interaction table with all four participation modes is present
- [x] Feature is described as opt-in, disabled by default

**Dependencies:** Task 1.1

**ADR Reference:** ADR-0010, Decision §1, §3, §5
