### Task 1.3: Add tech-writer dispatch to A-2    [medium]

**Description:** Add tech-writer dispatch logic to the A-2 Create section in `src/skills/author-adr/SKILL.md`. Between steps 3 (Pick a template) and 4 (Draft the ADR), insert conditional dispatch: if `tech_writer` is configured, dispatch it via the `task` tool; otherwise, the inline agent writes content as before.

**Implementation Notes:**

The ADR references "A-2 Step 3" for the integration point; in SKILL.md's actual numbering, "Pick a template" is step 3 and "Draft the ADR" is step 4. The dispatch logic goes into step 4 (Draft the ADR).

Insert dispatch logic into step 4 ("Draft the ADR"):

```markdown
4. **Draft the ADR** — populate the ADR body from the template.
   - **If `tech_writer` is configured** (non-empty value in `[author.dispatch]`):
     dispatch the tech-writer agent via the `task` tool with:
     - The ADR file path (with draft worksheet already populated from A-1)
     - The problem statement, constraints, and any context from the calling workflow
     - The template structure from [assets/templates/](assets/templates/)
     - Instructions to write Context, Options, Decision, Consequences, and
       Quality Strategy sections. Quality Strategy is a documentation task —
       the inline agent validates the selections during checkpoint review.
     If the configured agent cannot be resolved at runtime, fall back to inline
     writing and warn the user.
   - **If `tech_writer` is absent or empty** (default): the inline agent writes
     content directly, preserving current behavior.
```

## Stage 2: Reference Updates

### Task 2.1: Update create.md with tech-writer dispatch    [medium]

**Description:** Update `src/skills/author-adr/references/create.md` Step 3 ("Draft the ADR") to include the tech-writer dispatch logic. This is the deep reference that the agent reads during the full creation workflow. The dispatch logic here should be more detailed than the summary in SKILL.md A-2.

**Implementation Notes:**

In `references/create.md`, find Step 3 / Step 3a area. Add a new subsection `Step 3b: Tech-Writer Dispatch (Conditional)` after Step 3a (Draft Worksheet Integration):

```markdown
### Step 3b: Tech-Writer Dispatch (Conditional)

**Condition:** `[author.dispatch].tech_writer` is configured with a non-empty value.

When a tech-writer agent is configured, delegate the ADR body writing to it instead of writing inline.

1. **Build dispatch context** — assemble the payload:
   - The ADR file path (with draft worksheet from A-1 and any ASR/readiness context from Steps 1–2)
   - The problem statement, constraints, and stakeholder context from the calling workflow
   - The template structure (from `assets/templates/`)
   - Writing style instructions (from SKILL.md Writing Style section)
   - Section expectations: Context, Options, Decision, Consequences, Quality Strategy

2. **Dispatch via `task` tool** — invoke the configured tech-writer agent. The agent writes the ADR body content and returns control.

3. **Validate output** — after the tech-writer returns, verify:
   - All required sections are populated (Context, Options, Decision, Consequences)
   - Quality Strategy checkboxes reflect the decision's actual quality concerns
   - The content aligns with the draft worksheet's framing and constraints

4. **Fallback** — if the configured agent cannot be resolved at runtime, fall back to inline writing (the agent writes the content itself) and warn the user.

**When `tech_writer` is absent or empty:** Skip this subsection entirely — the inline agent writes content directly as the default behavior.
```

Also update the Step index table and procedure flow to include Step 3b.

- [x] Output validation step ensures completeness
