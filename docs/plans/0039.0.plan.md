## Stage 1: Schema & Model Changes

### Task 1.1: Create Diesel migration for `source_plan` column    [small]

**Description:**
Create a new Diesel migration that adds a `source_plan TEXT NOT NULL DEFAULT ''` column to the `task_summaries` table. The `DEFAULT ''` ensures existing rows retain valid data and JSONL records without the field ingest successfully.

**Implementation Notes:**
```sql
-- up.sql
ALTER TABLE task_summaries ADD COLUMN source_plan TEXT NOT NULL DEFAULT '';

-- down.sql
-- SQLite does not support DROP COLUMN in older versions.
-- For development tooling, a no-op down migration is acceptable.
```

Create the migration directory manually at `crates/adr-db/migrations/<timestamp>_add_source_plan/` with `up.sql` and `down.sql`. Then update `schema.rs` to include the new column.

**Test & Acceptance Criteria:**

- [x] `down.sql` exists (even if no-op)
- [x] `schema.rs` includes `source_plan` in the `task_summaries` table definition

## Stage 2: Producer & Consumer Updates

### Task 2.1: Update `extract-summary.awk` to emit `source_plan`    [small]

**Description:**
Modify `extract-summary.awk` to include a `source_plan` field in each JSONL line, derived from the input filename basename. Use awk's `FILENAME` variable, stripping the directory path to get just the basename (e.g., `docs/plans/0029.0.plan.md` → `0029.0.plan.md`).

**Implementation Notes:**
```awk
# In the output printf, add source_plan from FILENAME basename:
# Extract basename: split FILENAME on "/" and take the last element
n_parts = split(FILENAME, path_parts, "/")
basename = path_parts[n_parts]
```

**Test & Acceptance Criteria:**
- [x] JSONL output includes `"source_plan":"<basename>"` field
- [x] Basename is filename only (no directory path)
- [x] Update `extract-basic.expected` test fixture to include `source_plan`
- [x] `make test` passes (all extract-summary tests)
