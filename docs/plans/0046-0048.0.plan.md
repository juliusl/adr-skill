## Stage 2: solve-adr Branch Management (ADR-0046)

### Task 2.1: Update solve-adr SKILL.md with branch lifecycle    [medium]

**Description:** Add branch management instructions to the S-1 procedure in `src/skills/solve-adr/SKILL.md`. After S-1 Step 1 (intake), solve-adr creates a `solve/<problem-slug>` branch from current HEAD, switches to it, and all subsequent work happens there.

**Implementation Notes:**

Add to S-1 between intake and author steps:
- Branch creation: `git checkout -b solve/<slug>` after intake
- Dirty tree guard: check `git status --porcelain` before branching; warn if dirty
- Resume: check if branch exists with `git branch --list solve/<slug>`; if so, checkout and continue
- Branch collision: if branch exists but previous solve is complete (branch merged or deleted remotely), create `solve/<slug>-2` or prompt
- Branch name stored in conversation/session state for resume discovery
- Base branch: intentionally from current HEAD (document this is deliberate)

Do NOT add a new preference key — branching is default behavior.

**Test & Acceptance Criteria:**
- [ ] SKILL.md S-1 procedure includes branch creation step after intake
- [ ] Branch naming convention `solve/<problem-slug>` is documented
- [ ] Dirty working tree guard is documented
- [ ] Resume semantics are documented (checkout existing branch if unmerged)
- [ ] Branch collision handling is documented
- [ ] Base branch assumption (current HEAD) is explicitly stated as intentional
- [ ] No new preference keys added
- [ ] `make validate-solve` passes after changes
- [ ] `make check-refs` passes after changes

**Dependencies:** None

**ADR Reference:** ADR-0046, Decision (branch lifecycle, dirty working tree, resume)

### Task 3.2: Add user-mode support to author-adr scripts    [medium]

**Description:** Modify `nygard-agent-format.sh` and `new.sh` to support user-mode scope. When `ADR_SCOPE=user` is set, scripts resolve the ADR directory to `.adr/usr/docs/adr/` and prefix filenames with `$(whoami)-`.

**Implementation Notes:**

In `nygard-agent-format.sh`:
- Modify `resolve_dir()`: if `ADR_SCOPE=user`, return `.adr/usr/docs/adr` (creating with `mkdir -p` if needed)
- Modify the `new` subcommand: when scope is user, prefix the filename with `$(ADR_USERNAME:-$(whoami))-`
- Modify NNNN resolution: scan for files matching `$(username)-*.md` in the user directory, extract max NNNN, increment
- Modify `list` subcommand: when scope is user, list from user directory
- Modify `status` subcommand: when scope is user, operate on user directory

In `new.sh`:
- Pass `ADR_SCOPE` through to the format script

Username resolution: `ADR_USERNAME` env var if set, else `$(whoami)`.

**Test & Acceptance Criteria:**
- [ ] `ADR_SCOPE=user` causes scripts to use `.adr/usr/docs/adr/` directory
- [ ] Files created in user mode are prefixed with username (e.g., `juliusl-0001-title.md`)
- [ ] NNNN resolution is scoped to username-prefixed files in the user directory
- [ ] `list` in user mode shows only user-mode ADRs
- [ ] Scripts still work normally when `ADR_SCOPE` is unset (project mode)
- [ ] `ADR_USERNAME` override is respected when set

**Dependencies:** Task 3.1

**ADR Reference:** ADR-0047, Decision (convention, activation, username resolution)
