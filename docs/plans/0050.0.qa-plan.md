### Test-Gap Findings

- **TG1.1** — `make check-refs` only verifies file existence, not anchor validity. If headings are renamed to add identifiers (e.g., `## Good Practices` → `## P-1: Good Practices`), any internal `[text](#good-practices)` links within the same file or from SKILL.md will silently break. **check-refs does not catch this.** Mitigation: after restructuring, grep all `.md` files for anchor links (`](#...`) and verify each target heading exists.
- **TG1.2** — The dev plan criterion "behavioral semantics unchanged" relies on subjective diff review. No automated check validates that content was only reordered, not deleted. Mitigation: before restructuring, capture a word count and section heading list; after restructuring, compare word counts (should increase slightly from added index tables/guardrails, never decrease significantly) and verify all original headings are accounted for.
- **TG1.3** — manage.md uses a task-based pattern (M-1 through M-4) that is explicitly called out as uncertain in the ADR. The dev plan says "adapt the template" but no acceptance criterion validates that the adapted pattern is still recognizable as template-conformant. Mitigation: verify manage.md's task-selection routing diagram is functionally equivalent to a sequential flow diagram (the template's expected structure).
- **TG1.4** — Index table completeness is not automatically verified. A step could exist as a heading (`## Step 3: ...`) but be missing from the index table, or vice versa. Mitigation: after restructuring, extract all `## Step` / `## M-` / etc. headings and cross-reference against the index table rows. Every heading must appear in the index, and every index row must correspond to a heading.
