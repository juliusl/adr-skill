## Stage 1: Extend Revision Workflow with Addendum

### Task 1.1: Add Q&A addendum generation to revise.md Step 5    [medium]

**Description:** Update `references/revise.md` to add addendum-appending
behavior after the existing revision summary output. After Step 5 produces the
summary table, a new sub-step generates the Q&A addendum and appends it to the
ADR file. This is the core implementation of ADR-0016.

The addendum generation must:
1. Build Q&A entries from the revision items processed in Steps 2–3. Each
   finding becomes a `### Q:` heading (phrased as a question), and each user
   response becomes a `**Addressed|Rejected**` answer.
2. Detect whether the ADR already contains a `## Revision Addendum` section
   (by scanning for that heading). If found, append new entries below existing
   ones with a `<!-- Round N -->` marker. If not found, append `---` + the
   full addendum section.
3. Write the addendum to the ADR file using the edit tool.

**Implementation Notes:**

Insert a new step between current Step 5 and Step 6. The new step transforms
the revision items (already collected in Steps 2–3) into the addendum format
from ADR-0016 Decision § Format:

```markdown
---

## Revision Addendum

<!-- Generated by the revise task. Do not edit above the horizontal rule. -->

### Q: [finding phrased as question]

**Addressed** — [what was changed]

### Q: [next finding]

**Rejected** — [author's reason]
```

For existing addenda, skip the `---` and `## Revision Addendum` heading and
append only the new `### Q:` entries with a round marker.
