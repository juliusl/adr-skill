For each comment, offer the user three choices:

## Stage 2: Dispatch Hooks (ADR-0031)

Add configurable agent dispatch to the author-adr review→revise workflow
via `[author.dispatch]` TOML table with `review` and `editor` hooks.

### Task 2.1: Document [author.dispatch] config in SKILL.md    [medium]

**Description:** Add documentation for the `[author.dispatch]` table to the
SKILL.md Configuration section, after the existing `[author]` config docs.
Document:

1. The dispatch table schema: `review` (default: `"general-purpose"`) and
   `editor` (default: `"interactive"`) keys
2. The `"interactive"` reserved keyword meaning
3. The "same instructions, configurable executor" contract
4. Default behavior preservation (no dispatch table = current behavior)
5. Agent reference validation (graceful fallback on unresolved references)

Also add the dispatch table read to the Format Detection workflow step
(existing Step 1 reads `[author].template`; add a sub-step to read
`[author.dispatch]` keys).

**Implementation Notes:**

Add after the existing `### Project-Scoped Directory (.adr/)` section:

```markdown
### Agent Dispatch (`[author.dispatch]`)

Per ADR-0031, the review→revise workflow supports configurable agent dispatch.
Each hook point can be set to a specific agent or left at its default.

```toml
[author.dispatch]
review = "general-purpose"   # Agent for structured review (default)
editor = "interactive"       # Agent for editorial decisions (default: user)
```

| Hook | Default | Role | Instructions |
|------|---------|------|-------------|
| `review` | `"general-purpose"` | Reviewer | Receives `review.md` as prompt |
| `editor` | `"interactive"` | Editor | Receives `revise.md` as prompt |

The `"interactive"` value means "prompt the user directly." Any other value
is an agent reference. If a configured agent cannot be resolved, fall back
to the default and warn.
```

### Task 3.4: Update revise.md for editor worksheet consumption    [medium]

**Description:** Update `references/revise.md` to instruct the editor (human
or persona agent) to read the Draft Worksheet for intent grounding during
triage. Add an "Intent Grounding" subsection to the Editor Dispatch section
(added in Task 2.3).

Key content:
- The editor reads the Draft Worksheet for intent grounding during
  Address/Reject/Defer decisions
- Tolerance levels are **exploration calibration signals**, NOT finding triage
  policy — this is the critical calibration from ADR-0032's prototype
- Framing helps assess whether a finding aligns with the author's direction
- Uncertainty helps distinguish "the author knew about this" from "this is a
  gap the author acknowledged"

Include an explicit calibration warning:
```
⚠️ Tolerance levels control how much the agent diverges during OPTION
DISCOVERY. They do NOT control how aggressively the editor accepts or rejects
review findings. Low improvisation tolerance means "stay close to the framing
when exploring options," not "reject findings that challenge the approach."
```

Also update the Q&A append logic to work when a Draft Worksheet precedes the
Q&A entries in Comments. The existing `<!-- Generated by the revise task -->`
detection logic should append AFTER the worksheet, not at the start of
Comments.
