## Stage 1: Extraction Script & Tests

### Task 1.1: Create extract-summary.awk script    [small]

**Description:** Create an awk script at `src/skills/implement-adr/scripts/extract-summary.awk`
that reads a plan file from stdin or as a file argument. The script extracts the
implementation summary block (delimited by `<!-- BEGIN implementation-summary -->`
and `<!-- END implementation-summary -->`) and emits each task line as a JSONL
object to stdout.

**Implementation Notes:**
- Parse lines between the BEGIN/END markers
- Skip the header line (starts with `#`)
- Split each data line on ` | ` (pipe with surrounding spaces)
- Emit JSON: `{"task_id":"1.1","status":"done","cost":"small","commit":"abc1234","description":"..."}`
- Trim whitespace from each field
- Produce no output for plan files without a summary block (backwards compatible)

**Test & Acceptance Criteria:**
- [x] Script exists at `src/skills/implement-adr/scripts/extract-summary.awk`
- [x] Script is executable (`chmod +x` or invocable via `awk -f`)
- [x] Produces correct JSONL for a plan with a well-formed summary block
- [x] Produces no output for a plan without a summary block
- [x] Handles empty commit fields (task with no commit hash)

**Dependencies:** None
