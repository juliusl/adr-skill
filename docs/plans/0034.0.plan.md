### Task 1.1: Create wi-nygard-agent-format.sh with new and init subcommands    [medium]

**Description:** Create `src/skills/author-adr/scripts/wi-nygard-agent-format.sh` modeled on `nygard-agent-format.sh`. The script implements the `new` and `init` subcommands with work-item-referenced naming: `{remote}-{id}-{slug}.md` filenames and `# {remote}-{id}. {title}` headings.

**Implementation Notes:**

The `new` subcommand interface differs from `nygard-agent-format.sh`:
```bash
# nygard-agent: new <number> <title> <dir>
# wi-nygard-agent: new <remote> <id> <title> <dir>
```

Template reuses the nygard-agent template body but changes:
- Heading: `# {remote}-{id}. {title}` (e.g., `# gh-42. Use PostgreSQL`)
- Adds `Work-Item: {remote}#{id}` to the inline metadata block

Shared helpers (`slugify`, `resolve_dir`, `adr_date`, `parse_status`, `parse_title`) are identical — copy from `nygard-agent-format.sh`.

The `init` subcommand is identical to `nygard-agent-format.sh` — it bootstraps the ADR directory with `0001-record-architecture-decisions.md`.

**Test & Acceptance Criteria:**
- [x] `wi-nygard-agent-format.sh` exists and is executable
- [x] `new gh 42 "Use PostgreSQL" docs/adr` creates `docs/adr/gh-42-use-postgresql.md`
- [x] Generated file heading is `# gh-42. Use PostgreSQL`
- [x] Generated file contains `Work-Item: gh#42` in metadata
- [x] `init` creates `docs/adr/0001-record-architecture-decisions.md` (same as nygard-agent)

**Dependencies:** None

## Stage 2: Orchestrator and Makefile Integration

### Task 2.1: Update new.sh to support format-controlled naming    [medium]

**Description:** Modify `new.sh` to detect when the format handles its own naming (wi-nygard-agent). When `format=wi-nygard-agent`, skip sequential number computation and pass `remote` and `id` arguments from the command line instead.

**Implementation Notes:**

Current interface: `new.sh <format> <title...>`
New interface for wi-nygard-agent: `new.sh wi-nygard-agent <remote> <id> <title...>`

The orchestrator detects format-controlled naming by checking if the format script responds to a capability query, or more simply, by matching against known format names:

```bash
case "$format" in
  wi-nygard-agent)
    # Format handles its own naming — parse remote + id from args
    remote="${1:-}"; shift || true
    id="${1:-}"; shift || true
    title="$*"
    # Skip sequential number computation
    exec "$format_script" new "$remote" "$id" "$title" "$adr_dir"
    ;;
  *)
    # Sequential naming — compute next number
    title="$*"
    # ... existing logic ...
    exec "$format_script" new "$next" "$title" "$adr_dir"
    ;;
esac
```

**Test & Acceptance Criteria:**
- [x] `new.sh wi-nygard-agent gh 42 "Use PostgreSQL"` creates correct file via wi-nygard-agent-format.sh
- [x] `new.sh nygard-agent "Use PostgreSQL"` still works (backward compatible)
- [x] Error when remote or id missing for wi-nygard-agent format
- [x] Existing tests pass (`make test`)

**Dependencies:** Stage 1

**ADR Reference:** ADR-0034, Decision §Orchestrator Change

### Task 2.2: Update author-adr Makefile for wi-nygard-agent format    [medium]

**Description:** Update `src/skills/author-adr/Makefile` to support the wi-nygard-agent format. The Makefile is the primary downstream agent interface — its `new`, `rename`, and `status` targets currently assume sequential numbering (`$(NUM)`). Add `REMOTE` and `ID` variables for work-item-referenced operations.

**Implementation Notes:**

The Makefile needs format-aware targets. When `ADR_AGENT_SKILL_FORMAT=wi-nygard-agent`:
- `new` target: pass `$(REMOTE)` and `$(ID)` instead of computing next number
- `rename` target: use `$(REMOTE)` and `$(ID)` instead of `$(NUM)`
- `status` target: use `$(REMOTE)` and `$(ID)` instead of `$(NUM)`

The simplest approach: the format script already handles its own naming, so the Makefile passes the arguments through. For `wi-nygard-agent`, the `new` target invokes `new.sh wi-nygard-agent $(REMOTE) $(ID) $(TITLE)`.

**Test & Acceptance Criteria:**
- [x] `make new REMOTE=gh ID=42 TITLE="Use PostgreSQL"` creates correct file when format is wi-nygard-agent
- [x] `make rename REMOTE=gh ID=42 TITLE="Use MySQL"` renames correctly
- [x] `make status REMOTE=gh ID=42 STATUS=Accepted` updates status
- [x] Existing sequential targets still work when format is nygard-agent

**Dependencies:** Task 2.1

**ADR Reference:** ADR-0034, Quality Strategy (Tooling), Decision §Format Script
