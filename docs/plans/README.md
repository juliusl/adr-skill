# Implementation Plans

Plans generated by the `implement-adr` skill are stored here.

## Naming Convention

```
<adr-range>.<revision>.plan.md
```

- **adr-range:** Zero-padded ADR number(s) — `0003`, `0003-0004`, `0003-0004-0007`
- **revision:** Zero-based integer — `0` (initial), `1` (first revision), etc.

## Examples

| File | Description |
|------|-------------|
| `0003-0004.0.plan.md` | Initial plan for ADRs 3–4 |
| `0003-0004.1.plan.md` | First revision after user feedback |
| `0007.0.plan.md` | Initial plan for ADR 7 alone |
| `0003-0004-0007.0.plan.md` | Plan spanning non-contiguous ADRs |

## Revision Workflow

When changes are requested to a plan, a new revision file is created. The
previous revision is preserved unchanged. Each revision header links back to
its predecessor.

See [ADR-0005](../adr/0005-store-implementation-plans-in-docs-plans-with-versioned-naming.md)
for the full decision record.
