## Stage 1: Reference Document

### Task 1.1: Create `references/roadmap.md`    [medium]

**Description:** Create `src/skills/solve-adr/references/roadmap.md` — the self-contained reference document for the Roadmap scenario. This mirrors how `references/problem.md` serves S-1 Problem. The document defines the roadmap document format, the 6-step lifecycle (Load → Survey → Select → Solve → Update → Report), the resume protocol, the S-1 composition model, and defensive logging behavior.

**Implementation Notes:**

The document structure follows `problem.md`'s pattern:
1. Title + purpose statement
2. Key relationship (roadmap → milestones → S-1 → ADRs)
3. Resume protocol summary
4. Lifecycle diagram
5. Detailed step descriptions (Steps 1–6)
6. S-1 composition table
7. Defensive logging

The roadmap document format is derived from `docs/roadmaps/adr-atelier.md`:
- Project description paragraph
- `## Features` (optional)
- `## Constraints` (optional)
- `## Roadmap` with `### Milestone N` subsections
- Supplementary design sections below `---` separator

Progress tracking uses HTML comment status markers on milestone headings:
```markdown
### Milestone 1 <!-- status: complete -->
### Milestone 2 <!-- status: in-progress -->
### Milestone 3
```

Branch naming for roadmap milestones: `solve/<project-slug>/milestone-<N>`

**Test & Acceptance Criteria:**
- [x] File exists at `src/skills/solve-adr/references/roadmap.md`
- [x] Document is self-contained — usable without reading `problem.md`
- [x] Roadmap document format section matches `adr-atelier.md` structure
- [x] All 6 lifecycle steps are documented (Load, Survey, Select, Solve, Update, Report)
- [x] Resume protocol defined with milestone-level granularity
- [x] S-1 composition table clearly distinguishes S-2 orchestration from S-1 execution
- [x] Branch naming convention documented (`solve/<project>/milestone-<N>`)
- [x] Defensive logging section present (inherited from S-1 behavior)
- [x] Status marker format defined (HTML comments on milestone headings)
- [x] `make check-refs` passes for solve-adr (no broken references)
