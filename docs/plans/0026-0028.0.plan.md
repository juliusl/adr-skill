### Task 2.3: Implement `init` subcommand    [medium]

**Description:**
Replace the `init` placeholder with a working implementation that:
1. Resolves the database path (`.adr/var/adr.db` as default, overridable via `--db-path` flag)
2. Creates the `.adr/var/` directory if it doesn't exist
3. Runs embedded Diesel migrations against the SQLite database
4. Is idempotent — running `init` twice produces no errors
5. Follows Unix conventions: silent on success, errors to stderr

**Implementation Notes:**

Use `diesel_migrations::embed_migrations!()` at crate level. In the `init` handler:
```rust
use diesel::sqlite::SqliteConnection;
use diesel::Connection;
use diesel_migrations::{embed_migrations, EmbeddedMigrations, MigrationHarness};

pub const MIGRATIONS: EmbeddedMigrations = embed_migrations!();

pub fn run_init(db_path: &Path) -> Result<(), Box<dyn std::error::Error>> {
    if let Some(parent) = db_path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let db_url = format!("sqlite://{}", db_path.display());
    let mut conn = SqliteConnection::establish(&db_url)?;
    conn.run_pending_migrations(MIGRATIONS)?;
    Ok(())
}
```

Default path resolution: look for `.adr/var/` relative to current directory, or accept `--db-path` override.

**Test & Acceptance Criteria:**
- [ ] `adr-db init` creates `.adr/var/adr.db` (or specified path) with the schema applied
- [ ] Running `init` twice succeeds without errors (idempotent)
- [ ] `init` creates `.adr/var/` directory if it doesn't exist
- [ ] `--db-path` flag overrides the default database location
- [ ] Exit code 0 on success, non-zero on failure
- [ ] Silent on success (no stdout output), errors to stderr
- [ ] Integration test: `init` on a temp directory creates a valid SQLite database with the `task_summaries` table

## Stage 3: JSONL Ingestion

### Task 3.1: Implement `ingest` subcommand    [medium]

**Description:**
Replace the `ingest` placeholder with a working implementation that:
1. Reads JSONL from stdin, one record per line
2. Deserializes each line as `JsonlTaskRecord`
3. Inserts each record into the `task_summaries` table via Diesel
4. Reports errors per line to stderr (malformed JSON, missing fields) without aborting the stream
5. Accepts `--db-path` flag (same as `init`) to specify database location
6. Follows Unix conventions: silent on success, count of ingested records to stderr only if there were errors

**Implementation Notes:**

```rust
use std::io::{self, BufRead};

pub fn run_ingest(db_path: &Path) -> Result<(), Box<dyn std::error::Error>> {
    let db_url = format!("sqlite://{}", db_path.display());
    let mut conn = SqliteConnection::establish(&db_url)?;
    let stdin = io::stdin();
    let mut ingested = 0;
    let mut errors = 0;
    for (line_num, line) in stdin.lock().lines().enumerate() {
        let line = line?;
        if line.trim().is_empty() { continue; }
        match serde_json::from_str::<JsonlTaskRecord>(&line) {
            Ok(record) => {
                let new = NewTaskSummary { /* map fields */ };
                diesel::insert_into(task_summaries::table)
                    .values(&new)
                    .execute(&mut conn)?;
                ingested += 1;
            }
            Err(e) => {
                eprintln!("line {}: {}", line_num + 1, e);
                errors += 1;
            }
        }
    }
    if errors > 0 {
        std::process::exit(1);
    }
    Ok(())
}
```

**Test & Acceptance Criteria:**
- [ ] `echo '{"task_id":"1.1","status":"done","cost":"small","commit":"abc1234","description":"Test task"}' | adr-db ingest` inserts a record
- [ ] Multiple JSONL lines are ingested correctly
- [ ] Empty lines in stdin are skipped without error
- [ ] Malformed JSON lines produce an error message on stderr and continue processing remaining lines
- [ ] Exit code 0 when all lines ingested successfully, 1 when any errors occurred
- [ ] Silent on success (no stdout output)
- [ ] Unit test: ingest 5 valid JSONL lines, verify 5 rows in database
- [ ] Unit test: ingest mix of valid and invalid lines, verify valid ones are stored and error count matches
