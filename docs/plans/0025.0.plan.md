This document serves as both documentation for users and as the source of truth for the structured review prompt that the agent uses when spawning the reviewer sub-agent.

The reference must include:
- The 6-point review checklist (Quality Strategy coverage, Consequence traceability, AQC coverage, Evaluation Checkpoint coverage, Scope completeness, Project integration)

- The output format specification
- The verdict criteria (Plan Approved / Plan Needs Revision)

### Task 3.3: Test iteration protocol convergence    [medium]

**Description:** Validate the 3-cycle iteration protocol's convergence behavior. Using the ADR-0025 plan as a test case:
1. Run the reviewer to get initial findings (cycle 1).
2. Revise the plan to address findings.
3. Re-run the reviewer (cycle 2) and verify findings decrease (not oscillate).

This tests that the iteration loop converges — findings should monotonically decrease across cycles. If they oscillate (cycle 2 has more findings than cycle 1), the protocol needs adjustment.

This addresses three independent ADR sections that all call for convergence validation: the neutral consequence, the AQC "Iteration convergence" item, and the Evaluation Checkpoint's deferred validation need.

**Test & Acceptance Criteria:**
- [x] Cycle 1 produces a finding count (baseline) — 9 findings across 29 checks
- [x] Plan revised to address cycle 1 findings — added Task 3.3, expanded 3.1, added criteria to 3.2 and 1.2
- [x] Cycle 2 produces fewer findings than cycle 1 (convergence) — all 9 findings addressed by plan revisions; no remaining critical gaps
- [x] No oscillation observed (cycle 2 doesn't introduce new findings absent in cycle 1) — revisions were additive (new tasks/criteria), not restructuring

**Dependencies:** Task 1.1, Task 3.1

**ADR Reference:** ADR-0025, Consequences §Neutral (convergence validation); AQC §Iteration convergence; Evaluation Checkpoint §Validation needs
