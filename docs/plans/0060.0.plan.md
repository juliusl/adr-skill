### Task 1.2: Update C-1 substeps for multi-agent dispatch    [small]

**Description:** Update C-1a (entry condition), C-1c (dispatch), C-1d (triage), C-1e (re-review), and C-1f (gate) to handle a list of reviewers instead of a single reviewer.

**Files to update:**
- `src/skills/solve-adr/SKILL.md` — Conclusion section (lines ~237-288)

**Implementation Notes:**

- **C-1a:** Check if the normalized list is empty (skip) or non-empty (proceed)
- **C-1c:** Dispatch all agents in parallel via separate `task` tool calls. Each receives the same prompt. The dispatch template stays the same — each agent gets the full diff scope independently.
- **C-1d:** Triage consolidates findings from all reviewers into a single list. Deduplicate equivalent findings (same file, same issue, similar recommendation). When findings conflict, the triage actor resolves.
- **C-1e:** Re-dispatch all agents in parallel.
- **C-1f:** Gate checks all verdicts — if any reviewer says "Wait for Reviewer," the gate blocks.

**Test & Acceptance Criteria:**
- [ ] C-1a references "the normalized list" not "the configured agent"
- [ ] C-1c explicitly says "dispatch all agents in parallel via separate task tool calls"
- [ ] C-1d describes deduplication across multiple reviewer outputs
- [ ] C-1e says "re-dispatch all agents"
- [ ] C-1f says "if any reviewer" for the blocking condition
- [ ] Summary table (C-1 through C-1f) descriptions updated for plural
- [ ] `make check-refs` passes
