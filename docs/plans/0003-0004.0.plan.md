**Description:** Add a new step 7 to the "Creating an ADR" workflow in `author-adr/SKILL.md` that recommends reviewing the ADR with the `adr-reviewer` agent after creation. Includes the consent-based agent installation flow.

**Implementation Notes:**

After step 6 (Validate completion), add:
```markdown
7. **Recommend review** — after creating the ADR, recommend reviewing it with
   the bundled `adr-reviewer` agent:

   > Would you like to review this ADR with the adr-reviewer agent?

   If the user agrees:
   a. Check if the agent is already installed (look for `adr-reviewer.agent.md`
      in the project's agents directory).
   b. If not installed, ask the user for consent before installing:
      > The adr-reviewer agent needs to be installed as a custom agent.
      > This will copy `adr-reviewer.agent.md` to your agents directory.
      > Is that okay?
   c. If user consents, install via: `make -C <skill-root> install-agents`
   d. If user declines installation, fall back to the manual review process
      (inline ecADR checklist, fallacy scan, anti-pattern check).
   e. Run the reviewer agent on the newly created ADR.
```

### Task 2.2: Add interactive consequence validation to adr-reviewer agent    [medium]

**Description:** Extend `author-adr/assets/adr-reviewer.agent.md` to add a new step between the current Anti-Pattern Check (Step 3) and Review Checklist (Step 4) that interactively validates each consequence/pro/con assertion with the user.

**Implementation Notes:**

Insert a new Step 4 (renumber existing Step 4 → Step 5, Step 5 → Step 6):

```markdown
### Step 4: Consequence Validation (Interactive)

Review the Consequences section (Nygard) or Pros/Cons sections (MADR) with
the user to verify factual accuracy. For each stated consequence:

1. **Present the assertion** — quote the specific consequence.
2. **Assess plausibility** — flag assertions that appear speculative,
   unsubstantiated, or overly optimistic/pessimistic.
3. **Ask the user** — confirm whether the consequence matches their
   understanding of reality.
4. **Suggest revisions** — if inaccurate or unverified, propose revised
   wording that reflects the actual level of certainty.
```

Also add `ask_user` to the tools list in frontmatter since the agent needs interactive capabilities.
