## Stage 1: Unified Script Foundation

### Task 1.1: Create orchestrator script (new.sh)    [medium]

**Description:** Create `src/skills/author-adr/scripts/new.sh` — the
orchestrator script that handles common concerns for the `new` command:
resolving the ADR directory (`.adr-dir` or default `docs/adr`), computing the
next sequential ADR number, slugifying the title into a filename, and
delegating to the appropriate `<format>-format.sh` script.

**Implementation Notes:**

```bash
# Usage: new.sh <format> <title...>
# 1. Resolve ADR dir: read .adr-dir or default to docs/adr
# 2. Ensure directory exists (mkdir -p)
# 3. Find highest existing number, increment
# 4. Slugify title: lowercase, replace non-alnum with hyphens
# 5. Exec <format>-format.sh new <number> <title> <dir>
```

Key behaviors:
- Read `.adr-dir` file if present (search current dir only, not ancestors)
- Default to `docs/adr` if no `.adr-dir` exists
- Number format: 4-digit zero-padded (0001, 0002, ...)
- Slug: lowercase, spaces/special chars → hyphens, trim leading/trailing hyphens
- Error if format script not found or title not provided
- Support `ADR_DATE` env var pass-through (format script handles it)

**Test & Acceptance Criteria:**
- [ ] Script exists at `src/skills/author-adr/scripts/new.sh` and is executable
- [ ] Correctly resolves ADR directory from `.adr-dir` file
- [ ] Defaults to `docs/adr` when no `.adr-dir` exists
- [ ] Computes correct next sequential number (handles gaps)
- [ ] Slugifies titles correctly (spaces, special chars, case)
- [ ] Exits with error if format script is missing
- [ ] Exits with error if no title is provided

**Dependencies:** None

**ADR Reference:** ADR-0018, Decision §Architecture (Orchestrator)

### Task 1.2: Create nygard-agent format script    [heavy]

**Description:** Create `src/skills/author-adr/scripts/nygard-agent-format.sh`
— the self-contained format script for the nygard-agent ADR format. The
template from ADR-0017 is baked directly into the script. Implements four
subcommands via `case` dispatch: `new`, `init`, `list`, `status`.

**Implementation Notes:**

Baked-in template structure (from ADR-0017 Decision):
```markdown
# NUMBER. TITLE

Date: DATE
Status: Prototype
Last Updated: DATE
Links:

## Context
## Options
## Decision
## Consequences

## Quality Strategy
- [ ] Introduces major semantic changes
- [ ] Introduces minor semantic changes
- [ ] Fuzz testing
- [ ] Unit testing
- [ ] Load testing
- [ ] Performance testing
- [ ] Backwards Compatible

### Additional Quality Concerns

---

## Comments
```

Subcommands:
- `new <number> <title> <dir>` — generate ADR file from baked-in template
- `init [dir]` — bootstrap ADR directory, create first ADR, write `.adr-dir` if non-default
- `list` — list all ADRs with number, title, status; handle both inline `Status:` and `## Status` heading formats
- `status [num] [new-status]` — show/update status; handle both metadata formats

Backward compatibility: `list` and `status` must parse both inline `Status:`
metadata (nygard-agent format) and `## Status` heading (standard Nygard
format) for the transition period.

**Test & Acceptance Criteria:**
- [ ] Script exists at `src/skills/author-adr/scripts/nygard-agent-format.sh` and is executable
- [ ] `new` subcommand generates correct file with baked-in template
- [ ] Template includes all sections: Context, Options, Decision, Consequences, Quality Strategy, Comments
- [ ] Template uses inline metadata: Date, Status, Last Updated, Links
- [ ] `init` creates directory, writes first ADR, handles custom dir with `.adr-dir`
- [ ] `list` parses inline `Status:` format correctly
- [ ] `list` parses legacy `## Status` heading format correctly
- [ ] `status` shows status for a specific ADR (both formats)
- [ ] `status` updates status in inline `Status:` format
- [ ] `status` updates status in legacy `## Status` heading format
- [ ] `ADR_DATE` environment variable overrides date in generated files

**Dependencies:** None

**ADR Reference:** ADR-0017, Decision (template structure); ADR-0018, Decision §Architecture (Format script)

### Task 1.3: Create test suite for unified scripts    [medium]

**Description:** Create a test suite under `src/skills/author-adr/scripts/tests/`
with a Makefile test runner following the same diff-based validation pattern used
by the existing adr-tools and madr-tools test suites. Tests cover all four
subcommands: init, new, list, status.

**Implementation Notes:**

Test runner pattern (from existing suites):
- Each test is a `tests/<name>.sh` script + `tests/<name>.expected` output
- Makefile runs each test in an isolated `/tmp/` directory
- Sets deterministic `ADR_DATE` env var
- Compares output with `diff`

Required test cases:
1. `init-default` — init with default directory
2. `init-custom-dir` — init with custom directory and `.adr-dir`
3. `create-first-record` — create first ADR after init
4. `create-multiple-records` — create several ADRs, verify numbering
5. `list-records` — list ADRs showing number, title, status
6. `list-mixed-formats` — list ADRs with both inline and heading status formats
7. `show-status` — show status of a specific ADR
8. `update-status` — update status of an ADR

**Test & Acceptance Criteria:**
- [ ] Makefile exists at `src/skills/author-adr/scripts/Makefile` with `clean` and `check` targets
- [ ] At least 8 test cases covering init, new, list, and status
- [ ] Tests for backward compatibility (mixed inline/heading status formats)
- [ ] All tests pass (`make -C src/skills/author-adr/scripts clean check`)
- [ ] Test runner uses deterministic dates (`ADR_DATE=2026-01-15`)

**Dependencies:** Task 1.1, Task 1.2

## Stage 2: Build System Integration

### Task 2.1: Update author-adr Makefile    [medium]

**Description:** Rewrite `src/skills/author-adr/Makefile` to use the unified
scripts instead of the dual adr-tools/madr-tools backend. The
`ADR_AGENT_SKILL_FORMAT` variable defaults to `nygard-agent`. The Makefile
calls `new.sh` for the `new` target and `<format>-format.sh` directly for
`init`, `list`, and `status`. Remove the `generate` and `link` targets (dropped
per ADR-0018). Retain the `install-agents` target unchanged.

**Implementation Notes:**

Key changes:
- `ADR_AGENT_SKILL_FORMAT` default: `nygard` → `nygard-agent`
- Remove `ifeq ($(ADR_AGENT_SKILL_FORMAT),madr)` conditionals
- `new` target → `scripts/new.sh $(ADR_AGENT_SKILL_FORMAT) $(TITLE)`
- `init` target → `scripts/$(ADR_AGENT_SKILL_FORMAT)-format.sh init $(DIR)`
- `list` target → `scripts/$(ADR_AGENT_SKILL_FORMAT)-format.sh list`
- `status` target → `scripts/$(ADR_AGENT_SKILL_FORMAT)-format.sh status $(NUM) $(STATUS)`
- Remove: `generate`, `link`, `check-format` targets
- Keep: `help`, `install-agents`

**Test & Acceptance Criteria:**
- [ ] `ADR_AGENT_SKILL_FORMAT` defaults to `nygard-agent`
- [ ] `make -f <skill>/Makefile init DIR=docs/adr` works with unified scripts
- [ ] `make -f <skill>/Makefile new TITLE="Test Decision"` creates an ADR with nygard-agent template
- [ ] `make -f <skill>/Makefile list` lists ADRs correctly
- [ ] `make -f <skill>/Makefile status NUM=1` shows status
- [ ] `generate` and `link` targets are removed
- [ ] `install-agents` target still works

**Dependencies:** Stage 1

**ADR Reference:** ADR-0018, Decision §Commands retained; Consequences (Makefile targets simplified)
