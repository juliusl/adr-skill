## Stage 3: Add Subroutine Mode (ADR-0070)

Implement the core architectural change: S-1 gains a sub-routine mode that suppresses conclusion when called from S-2.

### Task 3.1: Add sub-routine mode to problem.md conclusion routing [small]

**Description:** Make problem.md's conclusion instruction conditional. Currently line 241 unconditionally says "proceed to the Conclusion sequence." Add a mode check.

**Current text:** "After implementation completes, proceed to the Conclusion sequence (C-1 → C-2 → C-3 → C-4) defined in SKILL.md."

**Fix:** Replace with:
```
After implementation completes:
- **Standalone mode (default):** Proceed to the Conclusion sequence (C-1 → C-2 → C-3 → C-4) defined in SKILL.md.
- **Sub-routine mode:** Return control to the caller. Do not run conclusion — the caller owns conclusion execution.

S-1 runs in sub-routine mode when the delegation prompt includes: "Run S-1 in sub-routine mode — skip conclusion, return control after implementation."
```
