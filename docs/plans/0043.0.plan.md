## Stage 1: Rename vendor → remote in ADR files

### Task 1.1: Rename in ADR-0034    [medium]

**Description:** Systematically replace all prefix-concept uses of "vendor" with "remote" in `docs/adr/0034-adopt-work-item-referenced-naming-for-adr-files.md`. Non-prefix uses (e.g., if "vendor-agnostic" appears as a design goal phrase) are unchanged per ADR-0043's Terminology Change table.

**Implementation Notes:**

Target replacements (prefix-concept uses):
- `{vendor}` → `{remote}` in naming convention patterns
- `vendor` field references → `remote` field
- Vendor prefix table entries (`gh`, `ado`, `local` rows) — change column header from "Vendor" to "Remote"
- `vendor-qualified` → `remote-qualified`
- `A local vendor` → update to use "remote" terminology
- Format script subcommand descriptions referencing `<vendor>`
- Cross-reference convention `ADR-{vendor}-{id}` → `ADR-{remote}-{id}`
- Draft Worksheet references to "vendor prefix scheme"

Do NOT change:
- "vendor-agnostic" if it appears as a design goal (describes a property, not the prefix)
- "vendor" in contexts that genuinely mean "a company/provider" not the prefix concept

**Test & Acceptance Criteria:**
- [ ] No remaining prefix-concept uses of "vendor" in ADR-0034 (`grep -i vendor` returns only non-prefix uses or zero results)
- [ ] All `{vendor}` patterns replaced with `{remote}`
- [ ] Naming convention table header says "Remote" not "Vendor"
- [ ] Format script subcommand table uses `<remote>` not `<vendor>`
- [ ] Cross-reference convention uses `ADR-{remote}-{id}`
- [ ] ADR title and heading are unchanged (the ADR describes the original decision; the rename is applied to its content)

**Dependencies:** None

**ADR Reference:** ADR-0043, Decision §Scope of Changes
