## Stage 1: Detection Logic (Rust)

### Task 1.1: Implement `detect_adapter()` in `author.rs`    [medium]

**Description:** Replace the `validate_remote()` function and `VALID_REMOTES` constant in `src/crates/adr-db/src/author.rs` with a `detect_adapter()` function. The function accepts a string that is either:
- `"local"` → returns `"local"` immediately
- Any other string → treated as a git remote name, runs `git remote get-url <name>`, matches the URL against known host patterns, returns the adapter type

URL pattern matching:
- Host contains `github.com` → `"gh"`
- Host contains `dev.azure.com` or `visualstudio.com` → `"ado"`
- No match → return error with the unrecognized URL and list of supported hosts

The function must handle both SSH URLs (`git@github.com:org/repo.git`) and HTTPS URLs (`https://github.com/org/repo`). Extract the host from either format before matching.

Replace all calls to `validate_remote(remote)` with `detect_adapter(remote)` across the five subcommands: `cmd_new`, `cmd_rename`, `cmd_status`, `cmd_lifecycle`, `cmd_export`. Each call site should use the returned adapter type (not the original user input) for filename operations.

Update the clap `/// Remote identifier (gh, ado, gitea, local)` doc comments on each subcommand's `remote` field to `/// Git remote name (e.g., origin) or 'local' for offline use`.

**Implementation Notes:**

```rust
fn detect_adapter(remote: &str) -> Result<String, String> {
    if remote == "local" {
        return Ok("local".to_string());
    }

    let output = std::process::Command::new("git")
        .args(["remote", "get-url", remote])
        .output()
        .map_err(|e| format!("failed to run git: {}", e))?;

    if !output.status.success() {
        return Err(format!(
            "git remote '{}' not found. Run 'git remote -v' to list available remotes.",
            remote
        ));
    }

    let url = String::from_utf8_lossy(&output.stdout).trim().to_string();
    detect_adapter_from_url(&url)
}

fn detect_adapter_from_url(url: &str) -> Result<String, String> {
    // Normalize: handle SSH (git@host:path) and HTTPS (https://host/path)
    let host = extract_host(url);
    if host.contains("github.com") {
        Ok("gh".to_string())
    } else if host.contains("dev.azure.com") || host.contains("visualstudio.com") {
        Ok("ado".to_string())
    } else {
        Err(format!(
            "Could not detect adapter type for URL '{}'. \
             Supported hosts: github.com, dev.azure.com. \
             For other forges, a future configuration mechanism \
             will allow explicit adapter mapping.",
            url
        ))
    }
}

fn extract_host(url: &str) -> String {
    // SSH: git@github.com:org/repo.git → github.com
    if let Some(rest) = url.strip_prefix("git@") {
        if let Some(host) = rest.split(':').next() {
            return host.to_lowercase();
        }
    }
    // HTTPS: https://github.com/org/repo → github.com
    if let Some(rest) = url.split("://").nth(1) {
        if let Some(host) = rest.split('/').next() {
            // Strip port if present
            return host.split(':').next().unwrap_or(host).to_lowercase();
        }
    }
    url.to_lowercase()
}
```

### Task 2.1: Implement `detect_adapter()` in `wi-nygard-agent-format.sh`    [medium]

**Description:** Replace the `validate_remote()` function in `src/skills/author-adr/scripts/wi-nygard-agent-format.sh` with a `detect_adapter()` function. The function accepts a string argument:
- If `"local"` → echo `"local"` and return 0
- Otherwise → run `git remote get-url "$1"`, extract host, match patterns, echo the adapter type
- If no pattern matches → print error to stderr and exit 1

Replace all calls to `validate_remote "$remote"` with adapter detection. Each call site should capture the returned adapter type and use it instead of the raw user input.

**Implementation Notes:**

```bash
# Detect adapter type from git remote name or 'local' keyword
# Echoes adapter type (gh, ado, local) to stdout
# Returns 1 on failure with error on stderr
detect_adapter() {
  local remote="$1"
  if [ "$remote" = "local" ]; then
    echo "local"
    return 0
  fi

  local url
  url=$(git remote get-url "$remote" 2>/dev/null) || {
    echo "ERROR: git remote '$remote' not found. Run 'git remote -v' to list available remotes." >&2
    return 1
  }

  detect_adapter_from_url "$url"
}

detect_adapter_from_url() {
  local url="$1"
  local host
  host=$(extract_host "$url")

  case "$host" in
    *github.com*) echo "gh" ;;
    *dev.azure.com*|*visualstudio.com*) echo "ado" ;;
    *)
      echo "ERROR: Could not detect adapter type for URL '$url'. Supported hosts: github.com, dev.azure.com." >&2
      return 1
      ;;
  esac
}

extract_host() {
  local url="$1"
  # SSH: git@host:path
  if echo "$url" | grep -q '^git@'; then
    echo "$url" | sed 's/^git@//; s/:.*//' | tr '[:upper:]' '[:lower:]'
    return
  fi
  # HTTPS: scheme://host/path
  echo "$url" | sed 's|.*://||; s|/.*||; s|:.*||' | tr '[:upper:]' '[:lower:]'
}
```

Each call site changes from:
```bash

```
to:
```bash

adapter=$(detect_adapter "$remote") || exit 1
```
Then use `$adapter` where `$remote` was previously used for filenames and metadata.
