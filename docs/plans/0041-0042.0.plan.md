## Stage 1: Project-Scoped Config Reading (ADR-0042)

### Task 1.1: Update prototype-adr SKILL.md Configuration section    [small]

**Description:**
Add project-scoped preferences documentation to the prototype-adr SKILL.md `## Configuration` section. After the user-scoped config paragraph, add:

```
**Project-scoped overrides:** If `.adr/preferences.toml` exists in the
project root, its keys override user-scoped settings. This allows
per-project experiment configuration without changing global preferences.
Each skill reads only its own namespace — prototype-adr reads `[prototype]`
keys and ignores others.
```

Update the config reading instructions in the Agent Workflow to read `.adr/preferences.toml` after user-scoped config.

**Test & Acceptance Criteria:**
- [x] SKILL.md Configuration section documents `.adr/preferences.toml` override
- [x] Config reading order is stated: user-scoped → project-scoped → defaults
- [x] Namespace isolation is documented (reads only `[prototype]` keys)
- [x] Missing `.adr/preferences.toml` is documented as a no-op (fall back to user-scoped)

**Dependencies:** None

**ADR Reference:** ADR-0042, Decision §Config reading order, §Skill changes

## Stage 2: Persona Experiment Protocol (ADR-0041)

### Task 2.1: Create `.adr/preferences.toml` with persona protocol    [small]

**Description:**
Create `.adr/preferences.toml` in the project root with the `[prototype.persona]` table:

```toml
[prototype.persona]
embed_source = true
ground_truth = "comments"
```

**Test & Acceptance Criteria:**
- [ ] `.adr/preferences.toml` exists with `[prototype.persona]` table
- [ ] `embed_source = true` is set
- [ ] `ground_truth = "comments"` is set

**Dependencies:** Task 1.1

**ADR Reference:** ADR-0041, Decision §Configuration

---

### Task 2.2: Document embed_source behavior in prototype-adr SKILL.md    [small]

**Description:**
Add a `### Persona Experiment Protocol` subsection under the Configuration section that documents the `embed_source` and `ground_truth` keys and their behavior:

- When `embed_source = true`: read ADR content up to `---` separator, embed in prompt, do not give file paths to the agent being tested.
- When `embed_source` is absent or false: default behavior (provide file paths normally).
- `ground_truth = "comments"`: indicates correct answers are in the Comments section after `---`.

**Test & Acceptance Criteria:**
- [x] SKILL.md has a `### Persona Experiment Protocol` subsection
- [x] `embed_source` behavior is documented (embed content, no file paths)
- [x] `ground_truth` key is documented
- [x] Default behavior (absent config) is documented

**Dependencies:** Task 2.1

**ADR Reference:** ADR-0041, Decision §Behavior when embed_source = true
