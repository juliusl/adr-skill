  mkdir -p .adr/var

### Task 1.3: Update `resolve_dir()` to check `.adr/adr-dir`    [small]

**Description:** Update `resolve_dir()` in `nygard-agent-format.sh` and the matching logic in `new.sh` to check `.adr/adr-dir` before falling back to `.adr-dir` and then the default `docs/adr`. This implements Convention 6 (supersedes `.adr-dir`).

**Implementation Notes:**

The priority order becomes:
1. `.adr/adr-dir` (new convention)
2. `.adr-dir` (legacy, still supported)
3. `docs/adr` (default)

```bash
resolve_dir() {
  if [ -f ".adr/adr-dir" ]; then
    cat .adr/adr-dir
  elif [ -f ".adr-dir" ]; then
    cat .adr-dir
  else
    echo "docs/adr"
  fi
}
```

The same logic must be mirrored in `new.sh`.

**Test & Acceptance Criteria:**
- [x] `resolve_dir()` reads from `.adr/adr-dir` when present
- [x] `resolve_dir()` falls back to `.adr-dir` when `.adr/adr-dir` is absent
- [x] `resolve_dir()` defaults to `docs/adr` when neither file exists
- [x] `new.sh` uses the same priority order
- [x] All existing tests still pass (backwards compatible)

**Dependencies:** None
