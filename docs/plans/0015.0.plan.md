## Stage 1: Revise Reference

### Task 1.1: Create `references/revise.md`    [medium]

**Description:** Create a new self-contained reference file at
`src/skills/author-adr/references/revise.md` documenting the full interactive
revise workflow. This file follows the same pattern as `references/review.md` —
it is loaded on demand by the agent when the revise task is activated.

The file must cover the 6-step workflow from ADR-0015:

1. Load review comments — parse the structured review output into discrete
   revision items (from completeness check, fallacy scan, anti-pattern check,
   consequence validation, and checklist sections).
2. Present each comment interactively — show the finding, relevant ADR section,
   and recommendation.
3. Collect user response — for each comment, allow "Address" (write revised
   text) or "Reject" (mark as rejected, with optional reason).
4. Apply revisions — apply accumulated changes to the ADR file.
5. Produce a revision summary — count addressed vs. rejected, list changes and
   rejections with reasons.
6. Recommend re-review — if substantive changes were made, suggest re-review.

The file should also document:
- How to parse the review output format (referencing `review.md`'s Output
  Format section)
- The revision summary output format
- Guard rails (e.g., don't modify sections the user didn't address)

**Implementation Notes:**

The review output follows a known structure (from `references/review.md`):
```markdown
## ADR Review: [title]
### Completeness (ecADR)
### Fallacies Detected
### Anti-Patterns Detected
### Consequence Validation
### Checklist
### Verdict: [Accept/Revise/Rethink]
```

Each section produces findings that become revision items. The revise reference
should instruct the agent to extract findings from sections that report issues
(not clean passes).

**Test & Acceptance Criteria:**
- [x] File exists at `src/skills/author-adr/references/revise.md`
- [x] File is self-contained — readable without needing other references
- [x] All 6 workflow steps from ADR-0015 Decision §Workflow are covered
- [x] Review output parsing guidance references the review.md output format
- [x] Revision summary output format is specified
- [x] Address and Reject response types are documented with behavior for each
- [x] File length is under 200 lines (141 lines)

**Dependencies:** None

**ADR Reference:** ADR-0015, Decision §Workflow, §Reference File
