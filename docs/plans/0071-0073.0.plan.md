### Task 1.2: Write version marker on install    [small]

<!-- Task execution protocol: Read criteria before starting. Mark checkboxes as satisfied. Verify all after completion. -->

**Description:** After a successful install, write a `.version` file in the skills directory and agents directory containing the binary's version string (`env!("CARGO_PKG_VERSION")` + `GIT_COMMIT_SHA`). This marker is read on subsequent installs to determine if an update is needed.

**Implementation Notes:**
```rust
fn write_version_marker(dir: &Path) {
    let version = concat!(env!("CARGO_PKG_VERSION"), " (", env!("GIT_COMMIT_SHA"), ")");
    fs::write(dir.join(".version"), version).unwrap_or_else(|e| {
        eprintln!("Warning: could not write version marker: {e}");
    });
}
```

### Task 1.3: Implement version check and dry-run logic    [medium]

<!-- Task execution protocol: Read criteria before starting. Mark checkboxes as satisfied. Verify all after completion. -->

**Description:** Before installing, read the existing `.version` file. If the installed version matches the embedded version and `--force` is not set, print "Already up-to-date" and return. If `--dry-run` is set, report what would be written (file count, target directory) and return without writing. If `--force` is set, skip the version check and install unconditionally.

**Implementation Notes:**
Decision logic at the top of `install_skills()` and `install_agents()`:
1. If `force` → proceed (current behavior: remove_dir_all + write).
2. Else if version file exists and matches → print "Already up-to-date (version: X)" and return.
3. If `dry_run` → count embedded files, print "Would install N files to <dir>", return.
4. Otherwise → proceed with install, write version marker.

**Test & Acceptance Criteria:**
- [ ] Version match + no force → prints "Already up-to-date" and exits without writing [implement]
- [ ] Version mismatch → proceeds with install [implement]
- [ ] No version file → proceeds with install [implement]
- [ ] `--dry-run` → reports file count and target, does not write files [implement]
- [ ] `--force` → always installs regardless of version [implement]
- [ ] Test: version match skips install [implement]
- [ ] Test: dry-run does not modify filesystem [implement]
- [ ] Test: force overwrites even with matching version [implement]

**Dependencies:** 1.1, 1.2

## Stage 3: Configurable Setup Defaults (ADR-0073)

### Task 3.1: Add flags to Init and Setup commands    [medium]

<!-- Task execution protocol: Read criteria before starting. Mark checkboxes as satisfied. Verify all after completion. -->

**Description:** Add CLI flags to `Init` and `Setup` for the most common customizations: `--participation` (guided/autonomous), `--auto-commit` (bool), `--auto-delegate` (bool), `--scope` (project/user), and dispatch hook overrides (`--tpm`, `--review`, `--tech-writer`, `--ux-review`, `--dx-review`, `--code-review`). Flags with no default use the embedded defaults.

**Implementation Notes:**
```rust

    /// Participation mode (guided or autonomous)
    #[arg(long, default_value = "autonomous")]
    participation: String,

    auto_commit: String,
    /// Enable auto-delegate to implement-adr
    #[arg(long, default_value = "true")]
    auto_delegate: String,
    /// Author scope (project or user)
    #[arg(long, default_value = "user")]
    scope: String,
    /// TPM agent name
    #[arg(long, default_value = "juliusl-tpm-v2")]
    tpm: String,
    /// Review agent name
    #[arg(long, default_value = "juliusl-editor-v5")]
    review: String,
    /// Tech writer agent name
    #[arg(long, default_value = "juliusl-tech-writer-v1")]
```
