**Implementation Notes:**

The SKILL.md structure follows the pattern from implement-adr and prototype-adr but uses scenario-based procedures:

| ID | Scenario | Mandatory | Description |

| S-0 | Startup | Yes | Load preferences, check config |
| S-1 | Problem Exploration | Yes | Problem → options → decision |

- Cross-skill invocation mechanism (skill tool delegation)
- Defensive logging principle
- Writing style (same as other skills)
